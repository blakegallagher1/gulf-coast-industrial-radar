# Investor Action Logic

Every alert should produce a concrete investor/developer action, not just a summary.

## Action Types

| Action | When It Applies |
|---|---|
| Map adjacent parcels | A site cluster appears near industrial infrastructure or a known facility |
| Identify owners | Land-control or site-fit signal emerges |
| Estimate assemblage value | Multiple related parcels are involved |
| Check zoning | Site appears industrially plausible but entitlement status is unclear |
| Check flood/wetlands | Site may be constrained by wetlands, flood zones, or coastal permitting |
| Call broker/owner | Evidence suggests early demand but ownership is still reachable |
| Monitor next board meeting | Agenda or public notice suggests a pending decision |
| Prepare option strategy | Related buyers or project entities appear active nearby |
| Pursue entitlement | Industrial use appears likely but not yet entitled |
| Pass | Source evidence is weak, late, overpublicized, or physically constrained |

## Alert Requirements

Each alert should include:

- What changed.
- Why it matters.
- Site/project/entity involved.
- Evidence links.
- Confidence score.
- Stage label.
- Recommended next action.
- Reason code for that action.
