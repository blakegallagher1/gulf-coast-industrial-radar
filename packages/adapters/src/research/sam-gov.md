# Source: SAM.gov Opportunities API v2
## Current production URLs
`https://api.sam.gov/prod/opportunities/v2/search` - Primary search endpoint for opportunities.

## Authentication & session model
Requires `api_key` query parameter (public API key obtained via SAM.gov registration). No session cookies or headers observed; key passed in every request.

## Endpoints / queries we use
- **GET** `/prod/opportunities/v2/search`
  - Required: `api_key`, `postedFrom` (MM/dd/yyyy), `postedTo` (MM/dd/yyyy, max 1 year range)
  - Optional (relevant to target fields): `noticeid` (for noticeId), `title`, `ptype` (type filter: o=solicitation, a=award notice, etc.), `state` (place of performance state), `naics` not directly filterable but returned in response, `solnum`
  - Example: `?api_key=KEY&postedFrom=01/01/2026&postedTo=01/31/2026&ptype=o&limit=25`

## Failure modes we should handle in the adapter
- Missing `postedFrom`/`postedTo` → 400
- Date range >1 year → invalid
- Invalid/missing `api_key` → 401/403
- "Description not found" for `description` link
- Empty `opportunitiesData` when `totalRecords` >0 (pagination edge)
